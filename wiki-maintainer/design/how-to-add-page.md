# How to Add a Concept or Tool Page (User Request)

When the user asks you to research a concept or tool and add a page for it, follow this process.

## 1. Clarify the concept or tool

Before doing anything else, confirm what exactly is being requested. If the name is ambiguous or covers several distinct things, ask the user to narrow it down. A well-scoped subject makes it possible to write a focused, accurate page.

## 2. Check if a page already exists

Read `index.md`. If a page for this concept or tool already exists, let the user know and offer to expand or update it rather than creating a duplicate.

## 3. Check for prerequisite pages

Consider whether there are closely related concepts or tools that don't yet have pages and that a reader would need to understand before this one. If any such pages are missing, create them first by following this same process, then return here.

Only proceed once any prerequisite pages are in the wiki.

## 4. Research and create the page

Follow `design/how-to-add-concept-or-tool.md` to research the subject and create the page.

## 5. Ensure coverage of referenced concepts and tools

Re-read the completed page. Enumerate every major concept and tool it mentions — these are the building blocks a reader would need to understand or follow up on the content.

For each one, check `index.md` to see whether a corresponding wiki page already exists. For any that are missing, follow this same process to create them.

Do this for every missing page before moving on. The goal is that every concept and tool named on the page is a live link a reader can follow.

## 6. Update the log

Append a log entry noting the new page and any other pages that were created or updated in the process.
